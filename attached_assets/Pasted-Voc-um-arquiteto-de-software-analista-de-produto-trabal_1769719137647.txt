ETAPA 3 — PÓS-CONCLUSÃO (VOLTA) SEM AFETAR PRÓXIMA IDA
=====================================================
Problema a resolver:
Após concluir a atividade, o técnico pode:
1) Voltar para BASE (registrar retorno)
2) Continuar para PRÓXIMA atividade (sem registrar retorno como ida)
3) Encerrar/ficar disponível

Regra crítica:
- O tempo de retorno (volta) NÃO é “tempo de ida” da próxima atividade.
- O tempo de ida da próxima atividade só será registrado quando o técnico clicar “Iniciar navegação” daquela próxima atividade.

PROPOSTA PRINCIPAL (recomendada no plano):
- Após concluir execução (e tratar sugestão RAT), mostrar um modal/painel “Próximo passo” com 3 ações grandes (TV/uso rápido):
  1) “Voltar para base”
  2) “Ir para próxima atividade”
  3) “Encerrar / Disponível”

Comportamentos:
1) VOLTAR PARA BASE
- Abrir GPS para a base do técnico (endereço/latlng do perfil).
- Capturar ETA (se possível).
- Abrir modal “Tempo de retorno à base” (igual ao de IDA):
  - mostrar ETA
  - tempo editável
  - tipo de deslocamento
- Salvar como registro separado: retorno_base (vinculado à atividade concluída + base_id).
- Não altera nada da próxima atividade.

2) IR PARA PRÓXIMA ATIVIDADE
- Não abrir modal de retorno.
- Apenas direcionar para a agenda (ou destacar a próxima).
- O tempo entre a atividade concluída e a próxima é “tempo de transição” não contabilizado por padrão.
- Quando ele clicar “Iniciar navegação” da próxima, roda a ETAPA 1 normal.

3) ENCERRAR / DISPONÍVEL
- Finaliza o fluxo, sem retorno.
- Opcional: setar status do técnico “disponível”.

ALTERNATIVA (para equipes que querem medir deslocamento entre atividades):
- Introduzir um tipo opcional “deslocamento_entre_atividades” (não é ida nem retorno_base).
- Mas por padrão, manter desligado (feature flag) para não gerar confusão.
- Essa alternativa deve ser descrita, mas NÃO recomendada como default agora (evitar complexidade).
